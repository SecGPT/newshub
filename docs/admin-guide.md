# 管理后台指南

管理后台位于 `/admin`，通过 `ADMIN_PASSWORD` 环境变量设置的密码登录。

## 认证

- 登录页：`/admin/login`
- 认证方式：密码验证后设置 Cookie（SHA256 哈希）
- 单管理员模式，无用户系统
- 所有 `/admin/*` 页面受布局级鉴权守卫保护

## 仪表盘（/admin）

展示系统概览：

- 分类、源、文章总数
- 各源的最近抓取状态（成功/失败/从未抓取）
- 近期错误列表

## 源管理（/admin/sources）

### 源列表

以表格形式展示所有 RSS 源：名称、Feed URL、所属分类、状态、最近抓取时间、文章数。

### 添加/编辑源

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | 源的显示名称 |
| Feed URL | 是 | RSS/Atom 地址 |
| 网站 URL | 否 | 源的官方网站 |
| 分类 | 是 | 下拉选择所属分类 |
| 刷新间隔 | 否 | 分钟数，0 = 使用全局默认 |
| 启用 | 否 | 是否参与定时抓取 |

### 测试 Feed

在添加前可验证 RSS URL 的有效性，预览最新 5 篇文章。

### 批量导入

支持 JSON 格式批量导入源：

```json
[
  { "name": "Hacker News", "feedUrl": "https://hnrss.org/newest", "category": "dev-cn" },
  { "name": "TechCrunch", "feedUrl": "https://techcrunch.com/feed/", "category": "科技资讯" }
]
```

`category` 字段支持分类 slug 或中文名称。

## 分类管理（/admin/categories）

- 列表展示所有分类，显示源数量
- 支持拖拽排序（调整 `sort_order`）
- 添加/编辑：名称、slug（自动生成）、图标、启用状态
- 删除前确认

## 系统设置（/admin/settings）

| 设置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 抓取间隔 | 数字 | 30 | 全局默认抓取间隔（分钟） |
| 报告时间 | 时间 | 08:00 | 每日报告生成时间 |
| 每源文章数 | 数字 | 10 | 首页每个源显示的文章数 |
| 保留天数 | 数字 | 30 | 自动清理超过此天数的文章 |
| 请求超时 | 数字 | 10 | 单个源抓取超时（秒） |
| 自动抓取 | 开关 | 开 | 是否启用定时抓取 |
| 自动清理 | 开关 | 关 | 是否启用过期文章清理 |

修改抓取间隔、报告时间等设置后，定时任务会自动重启以应用新配置。

## 定时任务

系统启动时自动运行三个定时任务：

| 任务 | 调度规则 | 说明 |
|------|----------|------|
| RSS 抓取 | 每 N 分钟（默认 30） | 并发抓取所有启用的源 |
| 日报生成 | 每天 HH:mm（默认 08:00） | 生成 Markdown 日报到 `output/` |
| 文章清理 | 每天 03:00 | 仅在开启自动清理时运行 |
